# 快速开始指南

## 🚀 5 分钟快速开始

### 前置条件
- JDK 17+ 已安装
- Maven 3.9+ 已安装
- 获取阿里云百炼 API Key（地址：https://bailian.console.aliyun.com/）

### 步骤 1：设置环境变量

**Windows PowerShell**：
```powershell
$env:AI_DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxxx"
$env:DASHSCOPE_MODEL="qwen3-235b-a22b-instruct-2507"
```

**Windows CMD**：
```cmd
set AI_DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxx
set DASHSCOPE_MODEL=qwen3-235b-a22b-instruct-2507
```

**Linux/Mac**：
```bash
export AI_DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxxx"
export DASHSCOPE_MODEL="qwen3-235b-a22b-instruct-2507"
```

### 步骤 2：启动应用

```bash
# 方式 1：使用 Maven 直接运行
mvn spring-boot:run

# 方式 2：使用已编译的 JAR
java -jar target/multi-agent-system-0.0.1-SNAPSHOT.jar
```

应用启动后，你将看到：
```
Started MultiAgentApplication in X.XXX seconds
```

默认监听端口：**8080**

### 步骤 3：启动 4 迭代 ADD 设计

**使用 curl**：
```bash
curl -X POST http://localhost:8080/api/v1/agents/architecture/design-iterations
```

**使用 PowerShell**：
```powershell
Invoke-WebRequest -Method Post http://localhost:8080/api/v1/agents/architecture/design-iterations
```

**使用 Postman**：
1. 打开 Postman
2. 新建 POST 请求
3. URL：`http://localhost:8080/api/v1/agents/architecture/design-iterations`
4. 点击 Send

### 步骤 4：查看结果

#### 控制台输出
应用会打印每个迭代的摘要：
```
========== 迭代 1 结果摘要 ==========
目标: 建立整体系统结构 - 定义顶层架构和核心模块
执行耗时: 45000 ms
...
```

#### 对话日志文件
完成后，查看：
```
./dialogue_logs/complete_dialogue_log.md
```

这个文件包含 4 个迭代的完整对话记录（可直接作为作业提交物）

## 🔧 故障排查

### 问题 1：API Key 无效
**错误信息**：`AI_DASHSCOPE_API_KEY 无效`

**解决**：
1. 确认 API Key 以 `sk-` 开头
2. 确认 API Key 所在的地域与模型一致
3. 在阿里云百炼控制台确认 API Key 未过期

### 问题 2：连接超时
**错误信息**：`Connection refused`

**解决**：
1. 确认应用已启动（看到 "Started MultiAgentApplication" 消息）
2. 确认端口 8080 未被占用
3. 确认网络连接正常

### 问题 3：模型不可用
**错误信息**：`Model not available`

**解决**：
1. 在 [百炼控制台](https://bailian.console.aliyun.com/) 确认已开通 `qwen3-235b-a22b-instruct-2507` 模型
2. 确认 API Key 有权限调用该模型

## 💡 使用建议

### 测试阶段
1. 先运行一次完整的 4 迭代
2. 查看 `./dialogue_logs/complete_dialogue_log.md` 检查输出质量
3. 如需调整，可修改：
   - `AddPromptTemplates.java` 中的系统指令
   - `application.yml` 中的模型参数

### 生成作业提交物
1. 运行系统生成对话日志
2. 复制 `./dialogue_logs/complete_dialogue_log.md` 作为"完整交互对话"提交物
3. 打包 `src/main/java` 作为"源代码"提交物
4. 编写报告分析架构设计结果
